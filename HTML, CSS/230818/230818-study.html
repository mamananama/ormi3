CSS

  • CSS: Cascading Style Sheet
  • CSS 스타일이 적용되는 우선순위가 폭포수 처럼 떨어지는 모양(Cascading)
  • HTML의 스타일, 레이아웃을 꾸미는 역할
  • CSS는 재사용성, 반복성을 가진다.

HTML 스타일 적용 방법론

  • 인라인 방식

    인라인 방식의 CSS는 스타일 적용 우선순위가 높으나, 코드의 가독성을 떨어뜨리는 문제를 일으킨다.

    일부 스타일 적용에 제한이 있음(예, 가상요소(:hover, ::before, ::after))

    <p style="color: royalblue;">안녕하세요 오르미 3기 여러분</p> <p>너무 너무 반가워요</p>

  • 내부 스타일

    head 태그 안 style 태그를 사용하여 스타일을 적용시킬 수 있음.

    코드 길이를 길어지게 만들어 코드 가독성을 떨어뜨리는 문제가 발생.

    로딩을 빠르게 적용시켜야 하는 일부 스타일의 경우 내부 스타일을 통해 작성하는 것이 이점이 있다.

  • 외부 스타일

    head요소의 link 태그를 통해서 외부 CSS 리소스와의 관계를 명시하여 스타일을 적용시킨다

    외부 스타일로 스타일을 적용하면 생산성과 코드 가독성이 높아진다.

CSS 상속

<div>
 <h1>Hello</h1>
 <h2>Hello</h2>
 <p>Hello</p>
 <button>Hello</button> </div>

  • 상속

    상속을 통해 부모의 속성 값을 자식에게 상속시킬 수 있다.

    위의 예에서, h1, h2, p는 부모인 div의 속성을 상속받는다.

    예외적으로, button 태그는 별도의 다른 속성을 지니기 때문에 원칙적으로 상속을 받지 않는다.
    대신, 'inherit' 속성 값을 통해 부모의 값을 상속받게 할 수 있다.

    'initial' 속성 값을 통해 브라우저 기본 스타일 속성을 따르게, 즉 상속을 받지 않게 할 수 있다.
